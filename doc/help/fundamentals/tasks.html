1 Introduction

A task is a piece of code that can be executed.

A task can have multiple attributes (or arguments, if you prefer).  The value of an attribute may contain references to a property.  These references will be resolved before the task is executed.

Tasks have a common structure:

<name attribute1="value1" attribute2="value2" ... />

where name is the name of the task, attribute# is the attribute name, and value# is the value for this attribute.

 For more information see the Task Reference.

2 Loading custom tasks

4.1 Implictly

When starting NAnt, the set of assemblies that will automatically be scanned for tasks can be devided into the following categories:

  1. OS-specific assemblies

    Depending on the OS, assemblies matching the following patterns will be scanned:

    Windows

    include

    *Tasks.dll
    *Tests.dll
    tasks/*.dll

    exclude

    NAnt.MSNetTasks.dll
    NAnt.MSNet.Tests.dll

    Unix

    TODO

  2. Runtime framework specific assemblies

    Depending on the runtime framework on which NAnt is running, some additional directories are scanned.

    In general, the following pattern applies:

    tasks/<framework family>/*.dll
    tasks/<framework family>/<framework version>/**/*.dll

    So, for .NET 1.0 you'd get the following pattern:

    tasks/net/*.dll
    tasks/net/1.0/**/*.dll

    Note:

    For all .NET framework versions the following additional patterns are also added to include a task assembly (and its corresponding test assembly) which is specific for Microsoft NET:

    NAnt.MSNetTasks.dll
    NAnt.MSNet.Tests.dll

Finally, whenever a project is started NAnt will scan the following directories for assemblies that match the *.dll pattern:

  • <Project Base Directory>\
  • <Project Base Directory>\<framework family>\
  • <Project Base Directory>\<framework family>\<framework version>

For a project file located in "c:\projects\log4net" with NAnt running on the Microsoft .NET 2.0 runtime, NAnt would end-up scanning the following project-level directories:

  • c:\projects\log4net
  • c:\projects\log4net\net\
  • c:\projects\log4net\net\2.0
